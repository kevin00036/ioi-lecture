直線求交點

  • $ \vec{d_1} = \vec{P_1P_2}, \vec{d_2} = \vec{P_3P_4}$
  • 交點 $ x = P_1 + t\vec{d_1} = P_3 + s\vec{d_2}$
  • $ t\vec{d_1} = \vec{P_1P_3} + s\vec{d_2}$
  • $ t\vec{d_1} \times \vec{d_2} = \vec{P_1P_3} \times \vec{d_2}$
  • $\displaystyle t = \frac{\vec{P_1P_3} \times \vec{d_2}}{\vec{d_1} \times \vec{d_2}}$

  • $\vec{d_1} \times \vec{d_2} \neq 0$ (為什麼?)

  • EPS : 誤差容忍值
  • 約 $10^{-6}$ ~ $10^{-14}$
  • 視題目性質而定

誤差傳遞

  • 有誤差的數字，經過運算後誤差會累積
  • $A = 1.0000\color{red}{2}, B = 1.0000\color{red}{1}$
  • $A+B = 2.0000\color{red}{3}$
  • $A \cdot B = 1.0000\color{red}{3}\color{gray}{0002}$
  • $A / B = 1.00000\color{red}{9}\color{gray}{9999\cdots}$
  • $A-B = \color{gray}{0.0000}\color{red}{1}$

誤差傳遞

  • EPS 要設夠大
  • 減法要特別注意

數據精度

  • EPS 要夠小，不能把不同的數誤判為相同
  • 輸入座標點為整數，範圍為 $[0, M]$
  • 不同點間相對誤差約 $O(1/M)$

數值方法

計算幾何中常會碰到求最小值的問題 :
給定一個函數 $f : [a, b] \to \mathbb{R}$ ，求 $\min f(x)$
... 別唬我了，這根本無法做吧 (圖 : 尖函數)

如果函數 $f$ 有一些條件呢？

如果一個函數 $f$ 在 $x_0$ 處有最小值，且在 $x < x_0$ 時嚴格遞減， $x > x_0$ 時嚴格遞增，則稱 $f$ 為嚴格單極值函數。
(圖)
  • 隨便戳一點 $x$
  • $f$ 在 $x$ 處遞減 : 極值在 $x$ 右邊
  • $f$ 在 $x$ 處遞增 : 極值在 $x$ 左邊
  • 二分搜！
  • 時間複雜度 : $O(T \log\left(\frac{b-a}{\epsilon}\right))$
  • $T$ : $f$ 求值一次的時間；$\epsilon$ : 誤差容忍值
(圖)
  • 如何判斷遞增或遞減？
  • 斜率 $f'(x)$
  • 取旁邊一個很近的點 $f(x+\epsilon)$ 比大小
  • 有時候不容易判斷

三分搜尋法

  • 戳兩點 $l, r$
  • $f(l) < f(r)$ : 極值在 $[a, r]$
  • $f(l) > f(r)$ : 極值在 $[l, b]$
  • $f(l) = f(r)$ ?
  • 時間複雜度 : $O(T \log\left(\frac{b-a}{\epsilon}\right))$
(圖)

多變數函數

  • 二維函數 $f(x, y)$
  • 二維的單極值函數，要怎麼找極小值？
  • $\displaystyle \min_{x, y} f(x, y) = \min_y \min_x f(x, y)$
  • 兩次三分搜
  • 時間複雜度 : $O(T \log^2\left(\frac{b-a}{\epsilon}\right))$

Gradient Descent

  • 真的只能亂走嗎?
  • 直接計算出函數值變小的方向!
梯度 (Gradient) : 概念上就是多維度的斜率。 梯度是一個向量，指向函數上升最快(坡度最陡)的方向； 大小即為該方向上的斜率。
$$\nabla f(x, y) = \left(\frac{\partial f}{\partial x}, \frac{\partial f}{\partial y}\right)$$

Gradient Descent

  • 每一次往負梯度的方向走一步
  • $(x, y) \gets (x, y) - \eta \cdot \nabla f(x, y)$
  • $\eta$ 參數決定了步長(走的速度)

步長的選擇

  • 步長太短 : 收斂太慢
  • 步長太長 : 會走過頭 (甚至不收斂)
  • 一開始步長大一點，接著慢慢縮小
  • 例如 : 每次乘以 $0.999$ 倍
  • 如果走得很順利，也可以放大
  • Trial & Error !

時間複雜度

  • $O(?)$
  • 看人品囉... 雖一點的話可能永遠走不完
  • 函數性質越好，通常會比較快收斂

凸函數

若一個函數 $f$，對於任意的 $x_1, x_2, 0 \le t \le 1$，皆符合 $$ f(t \cdot x_1 + (1-t)\cdot x_2) \leq t\cdot f(x_1) + (1-t)\cdot f(x_2)$$ 則稱 $f$ 是凸函數。
  • 1維凸函數：斜率單調遞增
  • 函數曲線永遠在兩點連線下方
  • 往每個方向走，斜率都單調遞增
  • Hessian Matrix 永遠是 Positive Semi-Definite
(圖)

凸函數

  • 凸函數一定單極值 (比單極值更強!)
  • 斜率單調
  • 越靠近極值點越平緩
  • 對 Gradient Descent 相當有利

常見的凸函數

  • $x^2$
  • $2^x$
  • $x^2 + y^2$
  • $\sqrt{x^2 + y^2}$ (直線距離)
  • 多個凸函數相加
  • 多個凸函數取$\max$

時間複雜度

  • ...還是$O(?)$
  • 但通常不會太差
  • 實際跑起來大概走個 $30$ ~ $1000$ 步左右會結束(通常啦...)
  • 保證時間複雜度? 函數條件要更強，而且要用較複雜的演算法
  • 最簡單的是牛頓插值法 (多維的也不好寫)
  • 其他的嘛... 「線性規劃只是一個特例」
  • 還是放棄吧XDDD

  • $N \ge 5$ : 沒有解析公式 Q__Q

Geometric Median

令 $$f(x, y) = \sum_{i=1}^N \sqrt{(x-x_i)^2 + (y-y_i)^2}$$ 求 $\min f(x, y)$。
  • $\sqrt{(x-x_i)^2 + (y-y_i)^2}$ 是凸函數
  • 凸函數相加還是凸函數
  • $f(x, y)$ 是凸函數!

Geometric Median

  • 用任何一個你喜歡的方法找極值
  • $\sqrt{x^2+y^2}$ 在原點有個尖點
  • $f(x, y)$ 在那 $N$ 個點處以外都是平滑的
  • 模擬退火 / Gradient Descent 都能順利找到極值

最小包含圓

平面上有 $N$ 個點，請找一個最小半徑的圓，把這 $N$ 個點包含在內。
(圖)

最小包含圓

  • 有各式各樣的算法
  • 枚舉3個點的外接圓 : $O(N^4)$
  • 隨機增量法 : 期望 $O(N)$
  • 數值方法

最小包含圓

設圓心為 $(x, y)$，需要的最小半徑即為最遠的點距離： $$f(x, y) = \max_{i=1}^N \sqrt{(x-x_i)^2 + (y-y_i)^2}$$ 求 $\min f(x, y)$。
  • $\sqrt{(x-x_i)^2 + (y-y_i)^2}$ 是凸函數
  • 凸函數取 $\max$ 還是凸函數
  • $f(x, y)$ 是凸函數!

最小包含圓

  • 取 $\max$ 有個小問題 : 函數會不平滑、有稜角 (不可微分)
  • Gradient Descent 會在兩壁間來回跳動
  • ...可能會吃土
  • 模擬退火比較容易成功

模擬退火 vs Gradient Descent

模擬退火:

  • 適用範圍比較廣
  • 甚至非單極值函數，也有機會找到最小值
  • 收斂比較慢

Gradient Descent:

  • 如果函數性質良好，收斂相當的快
  • 良好 : 凸函數、平滑函數 (可微分)
  • 如果函數很糟的話，可能會發散，或是收斂地非常非常慢

一些定義

一些定義

可以知道用 \( B \) 匹配 \( A \) 其實就是在問 \( B \) 是不是 \( A \) 的子字串！

Hashing

我們定義 Rolling Hash Function 為
$$ f(A) = a_0 p^{n-1} + a_1 p^{n-2} + \cdots + a_{n-2} p + a_{n-1} \mod{q} $$
其中 \( p, q \) 為挑好的質數

有點像進位制。

好處： 可以滾動(Roll)！

$$ f(A) \equiv f(A[0, n-2]) p + a_{n-1} \pmod{q} $$ 並且 $$ f(A[i, j]) \equiv f(A[0, j]) - p^{j-i+1} f(A[0, i-1]) \pmod{q} $$

記下所有 \( f(A[0, k]) \) ， 所有 \( f(A[i, j]) \) 都可立刻 ( \( \ord{1} \) )求出！

算出 \( f(A), f(B) \)

    → 不一定，但發生意外的機率極低！

碰撞的機率是 \( 1/q \) → 最好 \(q\) 大一點！

Fail function

在暴力法時我們每次匹配失敗時，往前一格，繼續匹配

但往前一格真的是必要的嗎？

其實有可能右移 \( 1, 2, \cdots, k-1 \) 都是不必要的，直接右移 \(k\) 就可以了。

Want: \( \fail(i) \) 要告訴我們，當我們匹配到 \( B[i] = A[j]\)，卻在 \( B[i+1] \) 的位置匹配失敗，應把 \( B[\fail(i)] \) 對齊 \(A[j]\) 繼續匹配。

\( \fail(0) \) 應該要是多少？

仔細思考一陣，你會發現如果 \( \fail(i) = k \) ， 那 \( B[0, k] = B[i-k, i] \)

而且 \( \fail(i) \) 還要是滿足此性質的最大質，否則會漏掉一些可能性

$$ \fail_B(i) = \begin{cases} \max \{ k: P_B(k) = B[0, k] = B[i-k, i] \} \\ -1 \\ \end{cases} $$

那要怎麼求出 \( \fail(i) \) 呢？

Fail function 的遞回

利用 \( \fail(k) : k \le i \) ， 我們可以求出 \( \fail(i) \) ！

不段嘗試 \( \fail(i), \fail^2(i), \fail^3(i), \cdots \)

Sample Code 


void build_fail_function(string B, int *fail) {
    int len = B.length(), current_pos;
    current_pos = fail[0] = -1; //Specially fail[0] = -1
    for( int i = 1 ; i < len ; i ++ ) {
        while( current_pos != -1
               && B[current_pos + 1] != B[i] ) {
            current_pos = fail[current_pos];
        }
        if( B[ current_pos + 1 ] == B[i] ) current_pos ++;
        fail[i] = current_pos;
    }
}

void match(string A, string B, int *fail) {
    int lenA = A.length(), lenB = B.length();
    int current_pos = -1;
    for( int i = 0 ; i < lenA ; i ++ ) {
        while( current_pos != -1 
               && B[current_pos + 1] != A[i] ) {
            current_pos = fail[current_pos];
        }
        if( B[current_pos + 1] == A[i] ) current_pos ++;
        if( current_pos == lenB - 1 ) {
            // Match ! A[i - lenB + 1, i] = B
            current_pos = fail[current_pos];
        }
    }
}
